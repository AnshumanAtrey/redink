# /redink-recipe — interactive section picker

Build the report recipe one section at a time. The user picks which of the 28 universal sections to include; the recipe is saved to `report-recipe.yaml` and read by `/redink-build`.

## Steps

### 1. Determine the starting point

- Read `framework.yaml` to find the active framework (or use `--framework <name>` if passed).
- Load `frameworks/<active>/manifest.yaml`. Its `sections` array is the **default selection** for this preset.
- Note any `explicitly_excluded_sections` (e.g. IRAP excludes `10-risk-rating-methodology`).
- If `--reset` was passed, ignore any existing `report-recipe.yaml` and start fresh.

### 2. Walk every section in the library

For each subdirectory in `sections/`, in numeric order (`01-` through `28-`):

1. Read `sections/<id>/section.md` frontmatter (title, required_by, inputs).
2. Decide the default:
   - If section ID is in active framework's `sections:` → default to **Y** (enabled).
   - If section ID is in `explicitly_excluded_sections:` → default to **N** (disabled, with a note).
   - Otherwise → default to **N**.
3. Print the section header + a 1-line description + the default + the question:

```
  ──────────────────────────────────────────────────────
  Section 01 — Cover Page
  ──────────────────────────────────────────────────────
  What it contains: Firm logo, title, engagement code, dates, contact.
  Required by every framework (universal).

  Default for <active framework>: ✓ ENABLED
  Include in this engagement?  [Y/n] _
  ──────────────────────────────────────────────────────
```

4. Capture the answer. Empty input takes the default.

### 3. Show summary + offer reorder

After walking all 28 sections, print:

```
RECIPE SUMMARY (15 sections selected)

   01-cover-page              ✓ enabled
   03-executive-summary       ✓ enabled
   04-scope-statement         ✓ enabled
   06-testing-methodology     ✓ enabled
   07-standards-adopted       ✓ enabled
   ...
   21-limitations             ✓ enabled
   22-references              ✓ enabled

   02-document-control        ✗ skipped
   05-out-of-scope            ✗ skipped
   ...

Total sections in final report: 15

Reorder sections? [y/N]
Save to report-recipe.yaml? [Y/n]
```

If reorder is requested, accept a comma-separated list of IDs in the new order.

### 4. Write report-recipe.yaml

Format:

```yaml
# Generated by /redink-recipe on YYYY-MM-DD
# Edit by hand or re-run /redink-recipe to regenerate.

framework_preset: certin-annexure-a
generated_at: YYYY-MM-DD HH:MM
total_sections_enabled: 15

sections:
  - { id: 01-cover-page,                enabled: true  }
  - { id: 02-document-control,          enabled: false }
  - { id: 03-executive-summary,         enabled: true  }
  - { id: 04-scope-statement,           enabled: true  }
  - { id: 05-out-of-scope,              enabled: false }
  - { id: 06-testing-methodology,       enabled: true  }
  - { id: 07-standards-adopted,         enabled: true  }
  - { id: 08-team-roster,               enabled: true  }
  - { id: 09-tools-used,                enabled: true  }
  - { id: 10-risk-rating-methodology,   enabled: false }
  - { id: 11-summary-of-findings,       enabled: true  }
  - { id: 12-key-observations,          enabled: true  }
  - { id: 13-aggregated-findings-table, enabled: true  }
  - { id: 14-per-finding-details,       enabled: true  }
  - { id: 15-controls-matrix,           enabled: false }
  - { id: 16-compensating-controls,     enabled: false }
  - { id: 17-poam,                      enabled: false }
  - { id: 18-threat-landscape,          enabled: false }
  - { id: 19-compliance-statement,      enabled: false }
  - { id: 20-remediation-roadmap,       enabled: false }
  - { id: 21-limitations,               enabled: true  }
  - { id: 22-references,                enabled: true  }
  - { id: 23-appendices,                enabled: false }
  - { id: 24-glossary,                  enabled: false }
  - { id: 25-acronyms,                  enabled: false }
  - { id: 26-signoff,                   enabled: false }
  - { id: 27-incident-reporting,        enabled: false }
  - { id: 28-continuous-monitoring,     enabled: false }
```

### 5. Confirm and exit

```
✓ Recipe saved to report-recipe.yaml
  Run /redink-build to assemble the report using this recipe.
```

## Flags

- `--framework <name>` — override `framework.yaml` for this recipe build
- `--reset` — ignore any existing `report-recipe.yaml`; start fresh from the framework defaults

## Notes

- Users who don't want interactivity can hand-edit `report-recipe.yaml` directly. The picker just makes the first run easier.
- If `framework.yaml` selects an `explicitly_excluded_sections` (e.g. IRAP excludes risk-rating), the picker warns when the user tries to enable it.
- A custom recipe (not matching any framework preset) is fine — just edit the file directly with whatever combination of sections you need.
